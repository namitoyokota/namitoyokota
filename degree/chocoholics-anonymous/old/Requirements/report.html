Use Cases

Manage Accounts

Use Case: Manage Member

Context: The software at the ChocAn Data Center is ran which allows operators to add new members to ChocAn, to delete members who have resigned, and to update member records.

Actors: Operator

Main Success Scenario:
 1. An operator is prompted to add, delete, or update member
 2. An operator selects their desired option
 3. The operator enters information of the member
 4. The operator submits the changes

Extensions:
 3a. The member does not exist in the system
     An error message is displayed on the screen then goes back to step 1
 3b. The member is suspended
     An error message id displayed and goes back to step 1
 3c. When adding a member, the member’s information already exists in the system
     The software alerts the operator then allow access to the provider to go to step 3

Verification

Use Case: Verify Service Code

Context: When the provider enters in the six-digit service code, the provider interacts with the system to verify the service code.

Actors: Provider

Main Success Scenario:
 1. The system finds the service name that corresponds to the service code.
 2. The system displays the service name to the terminal.
 3. The system prompts the provider to verify that the service name matches with the service that was provided.
 4. The provider confirms that the service name and service provided match.

Extensions:
 1a. A nonexistent service code was entered.
     1a.1 An error message is printed to the terminal.
     1a.2 The use case terminates.
 3a. The service name does not match with the service that was provided
     3a.1. The use case terminates.

Use Case: Complete Service Form

Context: The provider completes a form with information related to the service provided.

Actors: Provider

Main Success Scenario:
 1. The terminal displays a form that includes the current date and time, the date the service was provided, member name and number, service code, and fee to be paid to the terminal.
 2. The provider submits the completed form.

Healthcare Services

Use Case: Bill ChocAn

Context: The provider bills ChocAn for a service they recently performed.

Actors: Provider

Main Success Scenario:
 1. The provider recieves the member card from the member and passes the card through the reader.
 2. The word "VALIDATED" appears and the provider enters the date the service was provided.
 3. Using the provider directory, the provider looks up the appropriate six-digit code corresponding to the service provided.
 4. The provider keys in the service code.
     4a. The terminal communicates with ChocAn's Data Center to validate the service code.
 5. The software product writes a record to disk.
 6. The software product looks up the fee to be paid and displays it on the provider's terminal.
 7. The provider fills out a form in the terminal with the current date and time, the date the service was provided, member name and number, service code, and fee to be paid.

Extensions:
 1a. The member card is invalid.
     2a.1. The provider informs the member and the use case terminates.
 4a. The service code is invalid.
     4a.1. The terminal informs the provider and the use case continues from step 4.

Use Case: Verify Member extends Bill ChocAn

Context: The member recieves health care services from the provider through ChocAn

Actors: Member, Provider

Main Success Scenario:
 1. The provider recieves ChocAn member card from member.
 2. The provider slides the card through the ChocAn computer terminal's card reader.
 3. The terminal validates member number.
     3.1. The terminal communicates with the ChocAn data software to validate member number and membership.
 4. The provider provides health care service.
 5. The member receives an email with an attachment that lists services provided, sorted in order of service date, at the end of every week.
 6. The provider hands back the member card to the member.

Extensions:
 3a. The member is suspended.
     3a.1. The reason is displayed, member suspended.
     3a.2. Proceed to step 5.
 3b. The member number is not valid.
     3b.1. The reason is displayed, invalid number.
     3b.2. Proceed to step 6.

Accounting

Use Case: Execute Weekly Accounting Process

Context: At the end of each week on Friday at midnight the main accounting procedures run at the ChocAn Data Center

Actors: Scheduled Task (Timer)

Main Success Scenario:
 1. At midnight on Friday the main accounting procedure is run at the ChocAn Data Center.
     1.1. The weekly Member Report totals the provided services to each member.
     1.2. The weekly Provider Report totals the payable amount for services provided from each provider.
     1.3. The weekly Report for Accounts Payable totals the total payable amount for all services provided.
 2. The Funds Transfer Process is initiated and funds are electronically transferred.
 3. The ChocAn Systems publishes the reports and waits to be run again.

Extensions:
 1a. The main accounting procedure is run at the ChocAn Data Center by the request of a manager.
     1a.1 The use case continues at step 1.1
 1b. The ChocAn System is set to the time zone of the central location of the ChocAn Data Center.

Use Case: Produce a Report for Accounts Payable

Context: The ChocAn System produces a weekly summary report of accounts payable and sends it to the manager.

Actors: Manager

Main Success Scenario:
 1. The ChocAn System reads the week's files for all provider information related to billing, and consultations for services provided to members and creates an accounts payable summary report of all providers.
     1.1. The Report includes: every provider to be paid for the week, the number of consultation each had and related total fees, total number of providers who provided services, total number of consultations, and calculations of overall fee total.
 2. The ChocAn System sends the provider summary report to the manager.
 3. The manager receives the accounts payable report.

Extensions:
 1a. There is no new information for the current week related to billing, and consultations for services provided to members.
     1a.1 The ChocAn System creates a report stating that there is no new information related to billing, and consultations for services provided to members.
     1a.2The use case continues at step 2.

Glossary

Member : A user who receives service from ChocAn.

Member Report : A file for each member with name, number, address, and the service information for that week.

Manager : Handles Provider Report's and the Summary Report.

Member Record : A record of all ChocAn members in which operators can add new members, delete resigned members, and update member information.

Summary Report : a report given to the manager for accounts payable. Lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week

Provider : a health care professional who provides services to members

Provider Directory : a file created to provide an ordered list of service names and corresponding service codes and fees

Provider Record : A record of all ChocAn service providers in which operators can add, delete, or update provider information.

Provider Report : A weekly file for each provider containing that provider's personal information and consultation and service description for each service provided that week. The service description contains the total number of consultations provided and the total fee for the week.

Electronic Funds Transfer Record : A disk containing each provider's bank information and the amount to be paid.

Operator : Handles the Member and Provider Record's to add, delete, or update information.
