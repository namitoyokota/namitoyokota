Manage Accounts

Use Case: Manage Member

Context: The software at the ChocAn Data Center is ran which allows operators to add new members to ChocAn, to delete members who have resigned, and to update member records.

Actors: Operator

Main Success Scenario:
 1. The operator logs in to the system
 2. The system displays a menu for the operator who can select an option
 3. The operator chooses to manage members
 4. The system displays a menu with options to add, edit, or delete a member
 5. The operator performs the management process

Extensions:
 1a. The log in information is not valid
  1a.1. The system informs the operator
  1a.2. The use case goes back to step 1

Healthcare Services

Use Case: Bill ChocAn

Context: The provider bills ChocAn for a service they recently performed.

Actors: Provider

Main Success Scenario:
 1. The provider recieves the member card from the member and passes the card through the reader.
 2. The word "VALIDATED" appears and the provider enters the date the service was provided.
 3. Using the provider directory, the provider looks up the appropriate six-digit code corresponding to the service provided.
 4. The provider keys in the service code.
     4a. The terminal communicates with ChocAn's Data Center to validate the service code.
 5. The software product writes a record to disk.
 6. The software product looks up the fee to be paid and displays it on the provider's terminal.
 7. The provider fills out a form in the terminal with the current date and time, the date the service was provided, member name and number, service code, and fee to be paid.

Extensions:
 1a. The member card is invalid.
     2a.1. The provider informs the member and the use case terminates.
 4a. The service code is invalid.
     4a.1. The terminal informs the provider and the use case continues from step 4.

Use Case: Verify Member extends Bill ChocAn

Context: The member recieves health care services from the provider through ChocAn

Actors: Member, Provider

Main Success Scenario:
 1. The provider recieves ChocAn member card from member.
 2. The provider slides the card through the ChocAn computer terminal's card reader.
 3. The terminal validates member number.
     3.1. The terminal communicates with the ChocAn data software to validate member number and membership.
 4. The provider provides health care service.
 5. The member receives an email with an attachment that lists services provided, sorted in order of service date, at the end of every week.
 6. The provider hands back the member card to the member.

Extensions:
 3a. The member is suspended.
     3a.1. The reason is displayed, member suspended.
     3a.2. Proceed to step 5.
 3b. The member number is not valid.
     3b.1. The reason is displayed, invalid number.
     3b.2. Proceed to step 6.

Accounting

Use Case: Execute Weekly Accounting Process

Context: At the end of each week on Friday at midnight the main accounting procedures run at the ChocAn Data Center and produces reports

Actors: Scheduled Task (Timer)

Main Success Scenario:
 1. At midnight on Friday the main accounting procedure is run at the ChocAn Data Center.
 2. The ChocAn System generates a member report by reading the week’s file of services provided and creates a member report for each member.
  2.1. The report includes: Member Name (25 characters), Member number (9 digits), Member street address (25 characters), Member city (14 characters), Member state (2 letters), Member ZIP code (5 digits)
  2.2. For each service provided the report includes: Date of service (MM-DD-YYYY), Provider name (25 characters), Service name (20 characters)
  2.3. The ChocAn System sends the member report to each member via e-mail attachment.
  2.4. Each member receives their weekly service report.
 3. The ChocAn System generates provider report by reading the week’s files for each bill from providers and creates a provider summary report for each provider.
  3.1. The report includes: Provider name (25 characters), Provider number (9 digits), Provider street address (25 characters), Provider city (14 characters), Provider state (2 letters), Provider ZIP code (5 digits),
  3.2. For each service provided the report includes: Member name (25 characters), Member number (9 digits), Service code (6 digits), Fee to be paid (up to $999.99), Total number of consultations with members (3 digits), Total fee for week (up to $99,999.99)
  3.3. The ChocAn System sends the provider summary report to each provider via e-mail attachment.
  3.4. Each provider receives their weekly service report.
 4. The ChocAn System generates a summary reports by reading the week’s files for all provider information related to billing, and consultations for services provided to members and creates an accounts payable summary report of all providers.
  4.1. The Report includes: every provider to be paid for the week, the number of consultation each had and related total fees, total number of providers who provided services, total number of consultations, and calculations of overall fee total.   4.2. The ChocAn System sends the provider summary report to the manager.
  4.3. The manager receives the accounts payable report.
 5. The system uses the read-in files for each bill from providers to initiate the Electronic Fund Transfer Process.
  5.1. The record of electronic funds transfer data is written to a disk.
 6. The ChocAn Systems finishes the reports and waits to be run again.

Extensions:
 1a. The ChocAn System is set to the time zone of the central location of the ChocAn Data Center.
 2a. There is no new information related to services provided.
  2a.1 The ChocAn System skips over that member
  2a.2 The use case continues at step 2.1.
  3.a. There is no new information related to bill from providers.
   3.a.1 The ChocAn System skips over that provider.
   3.a.2The use case continues at step 3.1.
 4a. There is no new information for the current week related to billing, and consultations for services provided to members.
  4a.1 The ChocAn System creates a summary report stating that there is no new information related to billing, and consultations for services provided to members.
  4a.2 The use case continues at step 4.2.

Glossary

Member : A user who receives service from ChocAn.

Member Report : A file for each member with name, number, address, and the service information for that week.

Manager : Handles Provider Report's and the Summary Report.

Member Record : A record of all ChocAn members in which operators can add new members, delete resigned members, and update member information.

Summary Report : a report given to the manager for accounts payable. Lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week

Provider : a health care professional who provides services to members

Provider Directory : a file created to provide an ordered list of service names and corresponding service codes and fees

Provider Record : A record of all ChocAn service providers in which operators can add, delete, or update provider information.

Provider Report : A weekly file for each provider containing that provider's personal information and consultation and service description for each service provided that week. The service description contains the total number of consultations provided and the total fee for the week.

Electronic Funds Transfer Record : A disk containing each provider's bank information and the amount to be paid.

Operator : Handles the Member and Provider Record's to add, delete, or update information.
